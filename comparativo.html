Quadro comparativo de linguagens

Linguagem Pra que é usada? Nível de dificuldade Comentário
1° Python
  1. Web back-end;
  2. Aplicações desktop;
  3. Ciência de dados;
  4. Automação;
  5. Aplicações de deep learning;
  6. Machine learning e computação científica.
Baixo Código aberto, compatível com Mac e Windows, comunidade em constante crescimento. Porém é mais lenta por ser uma linguagem interpretada, consome bastante memória não é ideal para desenvolvimento de aplicações móveis.
2° C#
  1. Aplicações web.
  2. Aplicações móveis.
  3. Desenvolvimento de jogos e desenvolvimento de softwares de realidade virtual.
  4. Aplicações Linux e Mac.
 Moderada Tempos de compilação e execução rápidos, grande variedade de frameworks, perfeita para todos os tipos de desenvolvimento Windows. No entanto, tem curva de aprendizado inicial mais difícil do que outras linguagens de programação e não se traduz bem entre plataformas
3° Javascript
  1. Desenvolvimento web front-end;
  2. Desenvolvimento de jogos;
  3. Aplicações web.
 Baixo Código aberto. JavaScript client-side é incrivelmente rápida e não exige compilação, funciona muito bem com outras linguagens de programação populares. Porém demonstra problemas de segurança, desempenho instável entre diferentes navegadores.
4° PHP
  1. Desenvolvimento web;
  2. Aplicações desktop;
  3. Programas de computador.
 Baixo Código aberto, bem estabelecida para desenvolvimento web, excelente suporte da comunidade – grande comunidade de desenvolvedores e muitos materiais de apoio, Todavia, há muitas questões de segurança e criar um site utilizando PHP é uma tarefa mais lenta do que utilizar outras linguagens de programação, tem menos ferramentas de debugging
5° Java
  1. Desenvolvimento móvel;
  2. Desenvolvimento de apps;
  3. Aplicações desktop;
  4. Aplicações web;
  5. Desenvolvimento de jogos;
  6. Machine learning e computação científica.
 Baixo Altamente versátil. Simples de aprender, escrever, compilar e debugar, frameworks aceleram o desenvolvimento de programas Java, várias APIs disponíveis para o desenvolvimento de aplicações segurança aprimorada, compatível entre diferentes plataformas, No entanto, possui tempo de execução lento e consome bastante memória.
6° Ruby
  1. Desenvolvimento web tanto back-end quanto front-end.
 Médio Código aberto, rápida de escrever e fácil de debugar, sistema de garbage collection eficiente, grande comunidade de usuários. Todavia é lenta, menos flexível e possui problemas multi-threading.
